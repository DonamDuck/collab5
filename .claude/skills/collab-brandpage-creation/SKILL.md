# 컨시어지 브랜드 소개서 제작

대표가 지인 로컬 업체에게 선물할 브랜드 소개서를 Claude가 직접 제작하는 방법론.
2026-07-21 캔앤코르크B에서 확립. 그 전에 아그레아블·두더지요가원·상왕제약 홍익점·상왕라이브러리 4곳에서 검증됨.

## 핵심 테제

**자동 크롤은 공식 문서(보도자료·매거진)에 의존하고, 사장님이 직접 쓴 글(인스타 캡션·바이오·예약 플랫폼 소개문)을 놓친다. 소개서 품질은 사장님 목소리를 얼마나 회수하느냐에 달렸다.**

따라서 절차는 항상: 위저드로 베이스 생성 → 사장님 목소리 리서치 → 보강 → 저장·검증.

## 1단계 — 베이스 생성 (제품 위저드)

prod `https://collab5.vercel.app/register` 위저드에서 진행:

1. 실제 상호·지역·업종 입력
2. 크롤 칩 전부 선택
3. 별표 4개 (브랜드 정체성 축)
4. 링크 확인 (인스타/홈페이지/지도)
5. 한줄·자세히 후보 중 최선 선택
6. 찾은 이야기 담기 → 저장

⚠️ **유료 크롤이므로 대표 승인 하에 1회만** 실행한다.

## 2단계 — 사장님 목소리 리서치 (브라우저 탭 + API)

우선순위 순서대로:

1. **인스타 프로필 바이오**: `instagram.com/{handle}` 로 navigate → `meta[name=description]`에 바이오 전문이 **로그인 없이** 들어있음 (팔로워 수, 운영시간, 슬로건). 실측 검증됨.
2. **인스타 개별 포스트**: 포스트 URL(`/p/XXXX/`)로 navigate → `meta[property="og:title"]`에 캡션 전문. 포스트 페이지 DOM의 `a[href*="/p/"]`에서 형제 포스트 링크 8개 정도 수확 가능 → 최근 포스트 순회. **피드 목록 자체는 로그인월이라 불가.**
3. **보도기사**: 크롤이 찾은 기사 URL을 WebFetch로 읽어 구체 사실(대표 메뉴, 이벤트명, 인용) 추출.
4. **네이버 지역검색/블로그 API**: 주소 확정·콜라보 연도 확인 등. 프로젝트 `.env.local`의 `NAVER_` 키 사용, curl로 호출.

⚠️ `blog.naver.com`은 브라우저·WebFetch 정책 차단 — **curl로 공개 본문만** 접근.

## 3단계 — 보강 작성 원칙

### 사실 규율
- **출처 있는 사실만** 쓴다. 창작 금지.
- 연도·매체명 미확인이면 비워두거나 "2025년"처럼 넓게 쓴다.
- 근거가 외부인 게시물 1건뿐이면 "~하고 있어요"(정기 운영) 대신 **"~하실 수 있어요"(가능형)**로 강도를 낮춘다.
- ⛔ **영업시간·휴무는 소개서에 쓰지 않는다** (2026-07-21 대표 확정). 시간의 정본은 네이버지도 링크가 담당한다. 리서치에서 확보한 시간 정보는 검증·질문용으로만 쓴다.
- ⛔ **유통기한 있는 정보 금지**: 구체 가격("23,000원"), 기간 한정("7/24까지"), 선착순 증정 등. 단 **패턴화는 허용** — "매달 기획전을 열고 있어요"⭕ / "7월 뽀모리 샴페인 행사"❌.
- **활동 승격 기준**: 활동에는 반복되는 것만 (정기 프로그램·대표 메뉴·상시 서비스). 일회성 이벤트는 콜라보나 소개된 곳으로.
- **충돌 시 사장님 글 승리**: 크롤(제3자 문서)과 사장님 직접 작성 텍스트가 충돌하면 사장님 쪽을 따른다.
- **작은 숫자 자랑 금지**: 팔로워 수 등은 자랑이 되는 규모일 때만 숫자 지표로 승격한다.

### 보이스 (정본: `collab5-Obsidian/2_Areas/브랜드-보이스.md`)
- **해요체.** 격식체 남발 금지, 장식 따옴표 남발 금지, 맞춤법 점검.
- **현재형 원칙 (§7, 2026-07-21 대표 확정)**: 지금도 계속되는 일·상태는 **현재형/현재진행형**으로 닫는다 (`~하고 있어요` / `~이어가고 있어요`). 과거형(`~했어요`·`~였습니다`)은 **정말 끝난 사건에만**.
  - ❌ "2013년에 시작했어요" → ✅ "2013년에 시작해 지금까지 이어가고 있어요"
  - 적용 우선 3섹션: **자세히 소개 · 활동 · 콜라보**
  - 콜라보는 한 항목 안에서 시제 분리: 협약·행사 등 *사건*은 과거형, 그 결과로 *지금 하는 일*은 현재형. (예: "2025년 협약을 맺고 10호점이 됐어요. 지금은 주민 누구나 쉬어갈 수 있는 공간을 열어두고 있어요.")
- **1인칭 사장 시점**: 주어는 저희/우리 또는 생략. 브랜드명을 문장 주어로 쓰지 않는다.
- **상투어 금지**: 믿습니다·응원합니다·빛나는·소중한·특별한·함께 성장 등 어디에나 붙는 미사여구 배제. 추상 응원이 아니라 **구체 사실로 닫는다**.
- **사장님 고유 표현을 원형 보존**: 애칭, 슬로건, "낮술·혼술 환영" 같은 원문 단어를 최대한 그대로 반영.

### 섹션 구성
| 섹션 | 내용 |
|---|---|
| 자세히 소개 | 2~3문단: 장면 → 사장님 목소리 (영업시간 금지) |
| 활동 | 3~5개: 대표 메뉴 / 정기 프로그램 / 이벤트 |
| 콜라보 | 유형 지정 |
| 소개된 곳 | 구체 사실 + 기사 링크 |
| 공간 | 특징 칩 + 커스텀 칩 (예: 테라스) |
| 키워드 | 10개, 사장님 언어 우선 |
| 협업 기대 | 이력+공간에서 유도한 1~2문장 |

## 4단계 — 저장·검증

- 저장 전 폼 전체 값을 DOM으로 **전수 확인** (input/textarea value, 칩 ✓ 상태).
- 저장 버튼은 read_page ref로 **실클릭** (JS 클릭 아님).
- 저장 후 `/m/{slug}` 재조회로 전 섹션 노출 확인.

### 브라우저 함정
- React 상태 배칭 때문에 한 JS 실행에서 여러 클릭하면 **마지막 것만 반영** — 칩/키워드는 호출당 1개씩.
- read_page ref는 **뷰포트 내만 유효**.
- 백그라운드 탭은 screenshot으로 강제 페인트 후 조작.

### 보안·보고
- **비밀번호는 절대 에이전트가 입력·저장하지 않는다** — 로그인된 대표 계정 세션에서 작업하거나 대표가 직접 입력.
- 완성 후 대표에게 **미확인 항목(연도, 정기 여부 등) 질문 목록을 보고**한다.
