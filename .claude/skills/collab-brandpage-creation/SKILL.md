# collab5 브랜드 소개서 제작 — 2모드

대표가 지인 로컬 업체에게 선물할 브랜드 소개서를 Claude가 제작하는 방법론.
2026-07-21 캔앤코르크B에서 확립. 아그레아블·두더지요가원·상왕제약 홍익점·상왕라이브러리에서 검증됨.

## ⭐먼저 — 어느 모드인가 (필수)

두 가지 모드가 있고 **작업 시작 전에 반드시 어느 쪽인지 정한다.**

| | **모드 A — 순수 제품 생성** | **모드 B — 컨시어지 보강** |
|---|---|---|
| 정의 | **우리 웹 기능만으로** 생성. 사람 리서치 0. | 제품 생성 + Claude가 브라우저로 사장님 목소리 리서치 보강 |
| 별도 탭 크롤·기사검색·캡션 수집 | ❌ 안 한다 | ✅ 한다 |
| 손대는 것 | 위저드 선택 + 프리필 확인 + 저장. 위저드가 준 것만. | 위 + 2단계 리서치로 빈 섹션 보강 |
| 결과물 의미 | "사장님이 우리 사이트에서 혼자 만들면 나오는 것" = **제품 품질 실전 샘플** | 사람 손이 닿은 최대 품질 = **선물 최적화** |
| 언제 | 기능 검증·양산·다수 일괄 제작 | 특별히 공들일 소수, 자동 결과가 빈약할 때 |

**기본 규칙: 대표가 모드를 명시하지 않았으면 시작 전에 물어본다** —
"이거 순수 제품 기능만으로 만들까요(모드 A), 아니면 사장님 목소리까지 리서치해서 보강할까요(모드 B)?"
`/collab-ab-test`의 A/B 실험과는 다른 개념이다(그건 품질 측정용 2버전 비교, 이건 제작 방식 1택).

→ **모드 A면 "2단계 — 사장님 목소리 리서치"를 통째로 건너뛴다.** 1단계 생성 후 프리필 확인 → 바로 4단계 저장.
   단 "위저드가 준 press 링크를 저장에서 누락하지 않기"는 A에서도 지킨다(추가 수집이 아니라 누락 방지).
→ **모드 B면 아래 전체 절차를 따른다.**

## 핵심 테제 (두 모드 공통 배경)

**소개서 품질은 사장님 목소리를 얼마나 회수하느냐에 달렸다.** 자동 크롤은 원래 공식 문서(보도자료·매거진)에 의존하고 사장님이 직접 쓴 글(인스타 캡션·바이오·예약 플랫폼 소개문)을 놓쳤다.

⚡ **2026-07-22 업데이트 — 이 격차의 상당 부분이 제품에 코드화됐다**(`b7277b4`~`719ead3`).
이제 위저드가 자동으로 인스타 바이오·보도기사 본문을 읽고, 활동 0~5건·협업 항목까지 채운다.
따라서 **이 스킬의 역할은 "전부 수동 작성"이 아니라 "자동이 못 가져온 것만 메우기"로 좁아졌다.**

| 재료 | 지금 상태 |
|---|---|
| 인스타 **바이오** | ✅ 자동 (서버가 봇 UA로 fetch) |
| 보도기사 **본문** | ✅ 자동 (`fetchArticleExcerpts`) |
| 홈페이지 | ✅ 자동 (딥리드) |
| 인스타 **게시물 캡션** | ⚠️ **반자동** — 읽기는 자동이나 게시물 URL '발견'이 병목. **여기가 사람(브라우저)의 남은 핵심 가치** |
| 예약 플랫폼 소개문·지면 매체 | ❌ 수동 |

절차: 위저드로 베이스 생성 → **격차 확인** → 사장님 목소리 리서치(주로 캡션) → 보강 → 저장·검증.

### 폼에 자동으로 채워지는 것 (2026-07-22 기준)

저장 전에 아래가 이미 채워져 있다 — **덮어쓰지 말고, 빈 곳·틀린 곳만 손댄다.**
`✨ 미리 채웠어요` 배지가 AI 초안 표시다.

상호·한줄·자세히 / **활동 최대 5건**(대표 메뉴·정기 프로그램 등 '반복되는 것') /
콜라보 이력 / 소개된 곳(+기사 링크) / 공간 / 키워드 /
**함께하고 싶은 콜라보 칩(필수)** + **"이런 콜라보를 제공할 수 있어요" 텍스트** / 상세주소·인스타.

→ **모드 A는 여기까지.** 프리필 확인이 끝나면 2단계를 건너뛰고 **4단계(저장)로 바로 간다.**
   위저드가 준 press 링크만 저장에서 누락 없이 담고, 없는 링크·빈 섹션은 그대로 둔다.
→ **모드 B는** 비어 있거나 사장님 목소리가 안 들리는 섹션을 2단계에서 보강한다
   (그래도 전 섹션이 충분하면 리서치 생략 — 과잉 개입 금지).

## 2단계 — 사장님 목소리 리서치 (⚠️모드 B 전용 · 브라우저 탭 + API)

우선순위 순서대로 (1번이 사람만 할 수 있는 일이라 가장 값지다):

1. ⭐**인스타 개별 게시물 캡션** — 서버가 자동으로 못 하는 유일한 축.
   - 브라우저로 아무 게시물 URL(`/p/XXXX/`) 하나에 navigate → `meta[property="og:title"]`에 캡션 전문.
   - **그 페이지 DOM의 `a[href*="/p/"]`에서 형제 게시물 링크 8개쯤 수확** → 순회하며 최근 캡션 확보.
     (이 링크들은 JS 렌더라 **서버 HTML엔 없다** — 그래서 자동화가 막힌 지점이고, 브라우저를 쓰는 사람이 뚫을 수 있다.)
   - **프로필 피드 목록 자체는 로그인월이라 불가.** 진입점은 검색으로 찾은 게시물 URL 1개면 충분.
2. **인스타 프로필 바이오**: `instagram.com/{handle}` navigate → `meta[name=description]`.
   *이미 서버가 자동 수집하므로 보통 불필요* — 생성물에 바이오 색채가 안 보일 때 대조용으로만.
3. **보도기사**: 자동 발췌가 실패했거나(로그로 확인) 지면 매체라 URL이 없을 때 WebFetch로 직접.
4. **네이버 지역검색/블로그 API**: 주소 확정·콜라보 연도 확인 등. `.env.local`의 `NAVER_` 키, curl.

### 인스타 meta 수확의 UA 규칙 (실측)

- **브라우저(navigate + javascript_tool)**: 그대로 meta가 읽힌다.
- **curl·서버 fetch**: 일반 브라우저 UA로는 200이 와도 **meta 태그가 없다.**
  `facebookexternalhit/1.1` 같은 **링크 미리보기 봇 UA일 때만** meta를 준다.
  (제품 서버가 이 방식을 쓴다 — `src/lib/instagram.ts`)
- Vercel 데이터센터 IP도 차단되지 않는다(실측 확인).

⚠️ `blog.naver.com`은 브라우저·WebFetch 정책 차단 — **curl로 공개 본문만** 접근.

## 3단계 — 보강 작성 원칙 (⚠️모드 B 전용 · 모드 A는 위저드 문장 그대로 둔다)

### 사실 규율
- **출처 있는 사실만** 쓴다. 창작 금지.
- 연도·매체명 미확인이면 비워두거나 "2025년"처럼 넓게 쓴다.
- 근거가 외부인 게시물 1건뿐이면 "~하고 있어요"(정기 운영) 대신 **"~하실 수 있어요"(가능형)**로 강도를 낮춘다.
- ⛔ **영업시간·휴무는 소개서에 쓰지 않는다** (2026-07-21 대표 확정). 시간의 정본은 네이버지도 링크가 담당한다. 리서치에서 확보한 시간 정보는 검증·질문용으로만 쓴다.
- ⛔ **유통기한 있는 정보 금지**: 구체 가격("23,000원"), 기간 한정("7/24까지"), 선착순 증정 등. 단 **패턴화는 허용** — "매달 기획전을 열고 있어요"⭕ / "7월 뽀모리 샴페인 행사"❌.
- **활동 승격 기준**: 활동에는 반복되는 것만 (정기 프로그램·대표 메뉴·상시 서비스). 일회성 이벤트는 콜라보나 소개된 곳으로.
- **충돌 시 사장님 글 승리**: 크롤(제3자 문서)과 사장님 직접 작성 텍스트가 충돌하면 사장님 쪽을 따른다.
- **작은 숫자 자랑 금지**: 팔로워 수 등은 자랑이 되는 규모일 때만 숫자 지표로 승격한다.

### 보이스 (정본: `collab5-Obsidian/2_Areas/브랜드-보이스.md`)
- **해요체.** 격식체 남발 금지, 장식 따옴표 남발 금지, 맞춤법 점검.
- **현재형 원칙 (§7, 2026-07-21 대표 확정)**: 지금도 계속되는 일·상태는 **현재형/현재진행형**으로 닫는다 (`~하고 있어요` / `~이어가고 있어요`). 과거형(`~했어요`·`~였습니다`)은 **정말 끝난 사건에만**.
  - ❌ "2013년에 시작했어요" → ✅ "2013년에 시작해 지금까지 이어가고 있어요"
  - 적용 우선 3섹션: **자세히 소개 · 활동 · 콜라보**
  - 콜라보는 한 항목 안에서 시제 분리: 협약·행사 등 *사건*은 과거형, 그 결과로 *지금 하는 일*은 현재형. (예: "2025년 협약을 맺고 10호점이 됐어요. 지금은 주민 누구나 쉬어갈 수 있는 공간을 열어두고 있어요.")
- **1인칭 사장 시점**: 주어는 저희/우리 또는 생략. 브랜드명을 문장 주어로 쓰지 않는다.
- **상투어 금지**: 믿습니다·응원합니다·빛나는·소중한·특별한·함께 성장 등 어디에나 붙는 미사여구 배제. 추상 응원이 아니라 **구체 사실로 닫는다**.
- **사장님 고유 표현을 원형 보존**: 애칭, 슬로건, "낮술·혼술 환영" 같은 원문 단어를 최대한 그대로 반영.

## 4단계 — 저장·검증

- 저장 전 폼 전체 값을 DOM으로 **전수 확인** (input/textarea `.value`, 칩 ✓ 상태).
  ⚠️textarea는 placeholder(회색 예시)와 value를 혼동하지 말 것 — 반드시 `.value`로 판정.
- 제출이 소리 없이 막히면 **빈 필수 항목**(상호·콜라보 칩)이 원인 — 화면이 그 섹션으로
  스크롤만 되고 메시지가 없다. 칩 ✓를 재확인할 것. (offersHint 프리필 이후엔 드물지만
  힌트가 null이면 여전히 빈다.)
- 저장 버튼은 read_page ref로 **실클릭** (JS 클릭 아님).
- 저장 후 `/m/{slug}` 재조회로 전 섹션 노출 확인. slug는 랜덤(`m-xxxxxx`)이라 **즉시 기록**.
- 파이프라인이 실제로 돌았는지 의심되면 Vercel 로그 판독 — 방법은 `collab-ab-test` 스킬
  §Vercel 로그 판독 참조 (`instagram digest`·`article-excerpts` 라인).
