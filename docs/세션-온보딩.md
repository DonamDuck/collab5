## 1. 프로젝트 한 줄
collab5 = 콜라보 카드 매칭 서비스. prod=`https://collab5.vercel.app`, 코드=`~/Desktop/collab5`(Next16+Tailwind v4+TS+Supabase). **정본 두뇌 = Obsidian vault** `~/Desktop/collab5-Obsidian/`.

## 2. 작업 방식 (리듬)
- **새 기능/큰 변경** → `brainstorming`(설계·승인) → `writing-plans`(태스크화) → **서브에이전트로 순차 실행**(태스크별 커밋), 묶음마다 대표 QA. HARD-GATE: 설계 승인 전 구현 금지.
- **QA 소수정(문구·레이아웃·버그)** → 바로 구현·검증·배포. 빠른 반복.
- **결정은 추천부터**: 옵션 나열 말고 "이걸로 가자" 제안 + 이유. 유저가 리다이렉트하면 반영.
- **한국어 대화**, 레드팀/깊은 사고는 영어로 해도 됨.

## 3. 커밋·배포 규율 (⚠️ 중요)
- **배포 = `git push`** (Vercel 자동배포). push 전 **반드시 `npx tsc --noEmit` + `npm run build` 통과** 확인.
- **공유 작업트리 주의**: 여러 세션이 같은 트리를 쓰면 `git commit -am`/`git add -A` 금지 → **내가 만진 파일만 `git add <경로>`** 후 커밋(상대 세션 미커밋 변경 휩쓸림 방지). → [[collab5-shared-worktree-commit]]
- 커밋 메시지 한국어 OK. 끝에 `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **2팀은 아예 별도 브랜치/워크트리 권장** → §6.

## 4. 절대 규칙 (제약)
- 🔑 **API 키를 절대 대신 입력하지 않는다.** 유저가 직접 넣음("난 네 키를 직접 못 만져").
- 🔒 `service_role` 키는 **서버 전용** — 절대 `NEXT_PUBLIC_`로 노출 금지.
- 🧪 **와이프(캔버스가든)의 실제 콜라보 파트너를 테스트에 동원하지 않는다.**
- 🗄️ Supabase 콘솔 작업(버킷·SQL·유저삭제)은 **유저가 직접** 실행. 나는 정확한 SQL/클릭 순서를 제시만.
- 📄 서브에이전트 출력 JSONL을 **shell로 읽지 않는다**(컨텍스트 오버플로).
- 💻 유저가 "코드 보고 싶다" → 채팅에 붙이지 말고 `open -a "Visual Studio Code" <경로>`.
- 🖼 시각물은 **인라인 위젯(show_widget)** 또는 파일 open — preview 스크린샷은 유저에게 안 보임.

## 5. 검증 방식
- 코드 변경 → `tsc --noEmit` + `npm run build`.
- 브라우저 확인 필요 시 → `.claude/launch.json`의 dev 서버(`collab5` 3001 / `collab5-verify` 3007)로 preview.
- prod 확인은 배포 후. **라이브 유저 PII(profiles 등) 무단 조회 금지**(자동모드가 막음) — 필요하면 유저에게 요청.

## 6. 병렬 세션 격리 = git worktree (2팀 셋업)
같은 트리 공유의 커밋 충돌을 없애려면 **워크트리로 분리**:
```bash
cd ~/Desktop/collab5
git worktree add ../collab5-team2 -b team2      # 별도 폴더 + team2 브랜치
```
- 2팀 세션은 `~/Desktop/collab5-team2`에서 작업 → 파일·커밋·dev서버 완전 격리.
- dev 포트도 분리(2팀은 `.claude/launch.json`에서 다른 포트, 예: 3011).
- 완료 시 **PR로 main 병합**: `gh pr create` 또는 `git push -u origin team2` 후 GitHub에서 PR.
- **두 세션의 공용 정본 = Obsidian vault**(레포 밖, 공유) → hook/mention으로 소통.

## 7. 팀 소통 (vault hook/mention)
- **hook** ("~팀에 알려줘") → INDEX `📢 알림`에 FYI 추가.
- **mention** ("~팀에 요청") → 대상 노트 `⚠️ 미읽 멘션`에 체크박스 + INDEX 갱신.
- 큰 파일 만지기 전 [[디자인-시스템]] §🆕 최신 변경분 확인.

## 🔮 향후 아이디어 (백로그)
- **hook/메시지 기반 무인 실행**: 세션이 `isRunning:false`(닫힘)여도, 큐에 들어온 hook/메시지를 **자동으로 깨워 처리**하게 만들기. 현재 로컬 세션은 열려 있어야만 실행됨(닫히면 프로세스 없음). 무인화하려면 **클라우드 예약 에이전트(routines/cron)** 또는 **원격 에이전트(cloud offload)** 경로 필요. → 반복작업은 `/schedule`, 대화형은 원격 제어(모바일 이어작업)로 우회. 완전 무인 대화형 개발은 아직 제약.
