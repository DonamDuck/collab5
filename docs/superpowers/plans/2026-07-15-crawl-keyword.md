# 크롤 → 키워드 선택 재설계 — 구현 계획 (2026-07-15)

설계 정본: `collab5-Obsidian/1_Projects/크롤-키워드-재설계.md` (아이데이션 + 레드팀 4R 수렴)
원칙: **Gemini 콜 0 추가**, 태스크당 커밋, tsc+build 통과 후 배포(=main push), UI는 대표 실기기 QA 게이트.

## Phase 1 — 크롤 백엔드 정확도 (UI 무변경, 안전 배포)
현재 `gather()`는 상호만으로 넓게 검색하고 `mentionsBrand`(브랜드 문자열 포함)만으로 거른다 → 동명 충돌에 blind.
- **T1.1** 지역/업종 동의어 정적 테이블 (`src/lib/regionSynonyms.ts`) — 광역↔구↔동, 업종 변형. 오프라인, 콜 0.
- **T1.2** `gather()` negative-region 재랭킹: 넓게 recall 유지 + 상충 지역 토큰 언급 문서만 감점/드롭. 지역명 없는 본인 후기는 무변경.
- **T1.3** 지도 진실 교차검증: `local`(지도) roadAddress 지역이 유저 입력 지역과 일치할 때만 앵커 신뢰. 불일치/0건 → 저신뢰 플래그.
- **T1.4** thin 신호-티어 판정 헬퍼(`researchTier`): A(홈피메타·확인인스타·지도일치) / B(재랭크 후 블로그≥2) / C(빈손). research 메모에 티어 주석.
- **T1.5** 실크롤 검증(프로젝트 내 임시 스크립트, `.env.local` 키) — 동명 케이스로 노이즈 감소 확인 → 스크립트 삭제. tsc+build → 커밋 → 배포.

## Phase 2 — 키워드 추출 백엔드 (추가 API, 미사용까지 안전)
- **T2.1** `KeywordChip` 타입 { text; section; factual: boolean } + 조사메모→칩 추출 메서드(`extractKeywords`) — 기존 research 재사용, 콜 0. 사실 게이트(숫자/방송/수상/대표명 = factual=true).
- **T2.2** `draftBoth` 별표 가중치 주입: focusKeywords에 star(≤3) 우선순위 + verbatim(그대로 넣기) 항목 → 생성 후 문자열 존재검증 재시도.
- **T2.3** 업종 스타터 키워드(빈손 폴백용) — 업종 문자열 기반 정적 추천칩. tsc+build → 커밋 → 배포.

## Phase 3 — 씨앗 화면 (UI)
- **T3.1** 홈 상호 → register 진입 시 지역·업종 필수 입력 + "「상호·지역·업종」으로 찾을게요" 에코백.
- **T3.2** 크롤 트리거를 새 씨앗으로 배선. 대표 QA.

## Phase 4 — 키워드 선택·확인 화면 (UI, EnrichWizard 대체)
- **T4.1** 선택 화면(넓게, "최대한 많이 골라주세요…많이 고를수록 더 정확") + 클러스터 + 직접추가.
- **T4.2** 확인·강조 화면(좁게, ⭐토글 캡3) + 사실 게이트 칩(기본 OFF) + 인스타/홈피 확인(기본 미선택).
- **T4.3** 빈손 배너("온라인에서 확인 가능한 정보를 최대한 찾아봤지만, 기본 정보만 수집할 수 있었어요. 나머지는 직접 작성해 주세요.").
- **T4.4** 생성 → 한 줄+장문 초안 자유편집 → 폼. 대표 QA → 배포.
