# 소개서 블록 모듈화 — 설계 (2026-07-13)

> 브레인스토밍(대표+1팀) 확정본. "브랜드 소개서와 브랜드페이지의 중간" 비전의 **1단계 = 내용 다양화(모듈형 블록)**.
> 2단계(표현/테마 다양화) = vault [[소개서-표현-다양화]] 백로그.

## 배경 · 페인

현재 9섹션 폼은 모든 브랜드에 동일 → ①결과물이 다 비슷함 ②실전 자료(숫자·리뷰·역량)로 부족 ③쓸 동기가 약함.
리서치(미디어킷·입점제안서·와디즈/텀블벅·컴퍼니덱·Beacons/GBP) 결과, 표준 양식들엔 있는데 우리에 없는 것 = 숫자 증거·리뷰/추천사·팀·프레스·공간 스펙·"상대가 얻는 것" 프레임.

## 핵심 결정

| 결정 | 내용 |
|---|---|
| 접근 | **A안 "코어 + 블록" 증축형** — 기존 9섹션 유지, 선택 블록 라이브러리 신설 |
| 메커니즘 | AI 크롤 개인화 추천 + 자유 블록 추가. **업종 선택 스텝 없음**(크롤이 내부 추정) |
| 사업자등록 | 요구하지 않음 |
| PDF 업로드 | **라이트만** — 분석 없이 첨부. 코어 위계(⑨)로 승격 |
| 순서 조정 | **블록끼리만** 위/아래 이동. 코어 순서는 고정(정보 위계 보호). 전면 자유배치 = 2단계 |
| 가격/단가 블록 | v1 제외 (step2 매칭 때 재논의) |
| capability(B2B 역량) 블록 | v1 제외 — ⑤와 경계 모호. custom으로 커버, 수요 보이면 승격 |

## 블록 라이브러리 v1 (6종)

진입점(폼): **"+ 브랜드의 이야기를 더 담아볼까요?"** (선택) — 누르면 카탈로그 펼침.

| type | 폼 카탈로그 라벨 | 안내 한 줄 | /m 섹션 타이틀 | 고유 필드 |
|---|---|---|---|---|
| `metrics` | 우리를 보여주는 숫자 | 팔로워, 월 방문, 누적 판매 등의 지표도 콜라보에 도움을 줄 수 있어요. | 우리의 숫자 지표에요 | `items:{label,value}[]` 최대 4 |
| `reviews` | 고객들의 이야기 | 고객의 반응을 공유해보세요. | 이런 이야기를 들었어요 | `items:{quote,source?}[]` 최대 3 |
| `team` | 만드는 사람들 | 콜라버 정보를 등록하면 더 가깝게 느껴질 수 있어요. | 이런 사람들이 만들고 있어요 | `intro` textarea (사진 선택) |
| `press` | 소개된 곳들 | 수상이나 언론, 방송에 나온 적이 있다면요. | 이런 곳에 소개됐어요 | `items:{title,year?}[]` 최대 5 |
| `space` | 우리의 공간 | 공간이 있다면, 그 자체가 매력이 돼요. | 우리의 공간을 소개해요 | `desc` + `features` 칩 (기본 제안: 대관 가능 · 클래스 진행 · 촬영 친화 · 주차 · 팝업 경험 · 쇼룸 — 복수 선택, 기존 키워드칩 패턴) |
| `custom` | 직접 만들기 | 하고 싶은 이야기로 섹션을 직접 만들어보세요. | 직접 쓴 제목 | `title` + `body`. **최대 2개** |

- 블록 공통(BlockBase): **`photos[]`(최대 3, PhotoGrid 재사용) + `links[]`(`{label?,url}` 최대 3)**. 편집 카드 하단 [사진 담기]·[링크 추가] 버튼.
- custom 외 각 블록 1회만 추가 가능. 추가된 블록은 카탈로그에서 비활성.
- 카피 확정 이력: 이모지 제거, 해요체, "콜라버" 용어 채택(대표 코인). 느낌표는 /m 타이틀 관례 따라 제외.

## 폼 UX (/register · 수정폼 동일)

1. **블록 존 위치** = 코어 ⑦(콜라보 경험) 뒤, ⑧(주요 고객) 앞.
2. 진입 한 줄 → 카탈로그 그리드(6종, 라벨+안내) → 선택 시 인라인 편집 카드 추가(④·⑦ 카드 패턴 재사용). 카드 우상단 = 삭제 + 위/아래 화살표.
3. **소개자료 첨부(코어)**: ⑨ 안에 "(선택) 이미 만든 소개 자료가 있다면 함께 담아드릴게요." + [PDF 올리기]. Storage 업로드, 10MB 제한.
4. **⑤ subtitle 리프레임**: "제공할 수 있는 협업을 자유롭게 작성해주세요." → **"파트너가 우리와 함께하면 뭐가 좋을지, 편하게 들려주세요."**

## AI 크롤 개인화 (blockHints)

- `EnrichOptions`에 `blockHints: {type, reason, prefill?}[]` 추가 — 기존 OPTIONS_SYSTEM 판정 호출에 얹음(**추가 API 호출 없음**).
- 판정: 공개 수치 발견→`metrics`(prefill ⭕) / 언론·수상→`press`(prefill ⭕) / 공간 운영 흔적→`space`(추천만) / 후기 풍부→`reviews`(추천만 — 인용 자동채움은 조작 위험, 기존 "자동채움 안 함" 정책 유지). `team`·`custom`은 추천 없음.
- **추천 최대 2개.**
- 폼 노출 = 기존 크롤힌트 배너 문법 재사용: 블록 존 상단 접힌 배너 "웹에서 보니, 이런 이야기도 담으면 좋겠어요" → 추천 블록 + 근거("인스타그램에서 팔로워 1.2만을 봤어요") + [이 내용으로 시작하기].
- 신뢰 원칙: 근거 항상 표기, 밑그림은 확인·수정 전제(보이스 헌법 "미리 채워놨어요, 확인해주세요" 노선).

## 데이터 모델 (types.ts)

```ts
interface BlockBase { photos: string[]; links: { label?: string; url: string }[] }
export type Block = BlockBase & (
  | { type: "metrics"; items: { label: string; value: string }[] }
  | { type: "reviews"; items: { quote: string; source?: string }[] }
  | { type: "team";    intro: string }
  | { type: "press";   items: { title: string; year?: string }[] }
  | { type: "space";   desc: string; features: string[] }
  | { type: "custom";  title: string; body: string }
);
// Maker 추가 필드
blocks: Block[];        // 순서 = 배열 순서
introFileUrl?: string;  // 소개자료 PDF (코어 위계 — blocks 밖)
```

## /m 렌더

- 블록 존 = "함께한 콜라보" 뒤, "주요 고객" 앞 (폼 순서 = 소개서 순서).
- `metrics` = 스탯 타일(숫자 크게, 2열) — 유일한 시각 강조 블록 / `reviews` = 인용 카드 / `press` = 연도 리스트 / `team`·`space`·`custom` = 기존 Section 문법.
- 공통: photos = 섹션 하단 PhotoGrid, links = 링크 칩(신뢰뱃지 결).
- 내용 빈 블록 렌더 제외(기존 "값 없으면 숨김" 규칙).
- 소개자료 = 하단 [링크 복사] 옆 **[소개 자료 받기]**(있을 때만).

## 스키마 · 배선

```sql
alter table makers add column blocks jsonb not null default '[]';
alter table makers add column intro_file_url text;
```
- 유저가 Supabase 콘솔에서 직접 실행(운영 규칙). Storage 버킷 PDF mime 허용 1건.
- `repo.ts` 매핑 · `actions.ts` RegisterInput 확장 · 수정 모드 `blocks ?? []` 폴백. 검색·카드 무변경.

## 출시 3단계 (각각 독립 배포)

1. **블록 폼 + /m 렌더** — 카탈로그·편집카드·순서이동·렌더러
2. **소개자료 첨부** — PDF 업로드 + [소개 자료 받기]
3. **blockHints 개인화** — enrich 확장 + 추천 배너

## 검증 · 엣지

- 단계마다 `tsc --noEmit` + `npm run build` → `?demo=1` 프리필에 블록 샘플 → dev QA → 배포 → 캔버스가든 소개서 실사용 확인.
- 엣지: 빈 블록 제출 제외(⑦ 패턴) · 사진 업로드 중 제출 잠금(기존) · PDF 10MB 제한 · 기존 소개서 blocks 없음 → `[]`.
