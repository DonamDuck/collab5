# 소개서 미리보기 + 사진 UI 완화 — 설계 (2026-07-17)

## 배경 / 문제 (대표 실사용 피드백)
AI 크롤은 잘 끝났는데 **사진이 준비 안 된 유저**가 폼의 사진 첨부 UI 압박에 이탈하고 싶어함. 사진 없이 생성한 페이지를 보여주자 "오? 괜찮은데?" 반응.
1. 고객은 사진을 충분히 갖고 있지 않다.
2. 작성 시점에 어떤 소개서가 나올지 인지하지 못한다 (예시 부재).
3. 사진 첨부 UI가 모든 카드에 잘 그려져 있어(메인 10장·활동 3장·콜라보 3장·블록 3장) 사진 없는 고객이 지친다.

## 1. `/preview` 라우트 (신규)
- 탭 2개: **[사진 있는 소개서] / [사진 없는 소개서]**. 쿼리 `?tab=photo|none`으로 초기 탭 지정(기본 `photo`).
- 내용: 데모 소개서 2종을 기존 `/m/[slug]` 렌더 재사용으로 표시. 상단에 짧은 안내 1줄 + 탭, 하단에 "내 소개서 만들기 → /register" CTA 1개.
- **⚠️ 렌더 재사용은 공짜가 아님 (레드팀 CONFIRMED-1)**: `/m/[slug]`는 서버 페이지라 본문 렌더를 **공유 컴포넌트로 추출하는 리팩터링이 명시적 선행 태스크** — 핵심 제품 화면을 건드리므로 **QA 게이트 추가: 추출 후 실제 프로덕션 소개서 1종이 픽셀 동일하게 렌더되는지 확인 후에** /preview 작업 진행.
- `?embed=1`: 사이트 헤더·CTA 숨김(향후 임베드용 미니멀 모드).
- 데모 소개서는 `/preview` 전용 고정 slug 2개를 코드 상수로 참조. **데모 행이 없으면(실수 삭제 등) 크래시 대신 안내 문구 + /register CTA로 우아하게 처리** (레드팀 CONFIRMED-2b).

## 2. 데모 복제 (1회성 스크립트)
- `scripts/clone-demo-makers.ts`: 원본 2종(`m-ofjghi`=사진O, `m-ay6uve`=사진X)을 데모 행으로 복제.
  - 새 slug 확정: `m-demo-photo`, `m-demo-none`, **`search_visible=false`** (검색 제외 — 2팀 배포 필터 재사용), `collab_open=false`(콜라보 요청 유입 방지).
  - **사진·로고 Storage 객체도 데모 경로로 복사** 후 URL 교체 — 원본 사진 삭제 시 샘플이 깨지는 것까지 차단(완전 동결). 실사용자 앞에서 이미지 깨짐은 조용한 실패라 ~30줄 값어치 있음(레드팀 판단 동의).
  - 재실행 = 재복제(멱등: 기존 데모 행 갱신 + **데모 경로 기존 객체 정리·덮어쓰기** — 고아 파일 없음). 대표가 "샘플 갱신해줘" 하면 이 스크립트 재실행.
- 스키마 변경 없음. DB 행 2개 + Storage 사본만 추가.
- **⚠️ /my 오인 방지 (레드팀 CONFIRMED-2a)**: 클론이 대표 계정 소유라 **/my에 원본과 클론이 같은 이름으로 나란히 노출** → 실수 수정/삭제가 동결 전제를 깨뜨림. 해소: **/my 카드에서 slug가 `m-demo-` 프리픽스면 "🔒 미리보기 고정본" 배지 표시** (판별은 slug 기반 — 제목 오염 없음, 공개 `/m`·`/preview` 렌더에는 아무 표시 없음). 삭제 자체는 막지 않되(대표 권한), 삭제 시 /preview는 §1의 우아한 폴백으로 동작.

## 3. 홈 변경 (`src/app/page.tsx` — ⚠️ 핫파일, 착수 전 `git fetch`)
- 버튼 라벨 `예시 소개서 보기` → **`소개서 미리보기`**, href `/search` → `/preview`.
- 히어로 바로 아래·3스텝 위에 **미리보기 섹션**: 데모 소개서 정적 스크린샷(모바일 폰 프레임 느낌, 사진 있는/없는 나란히 — 모바일에선 세로 스택). `next/image` **lazy·non-priority**(폴드 아래) + 폭 제한 → LCP 게이트 무난 통과.
- 스크린샷 파일은 `public/preview/` 정적 자산으로 커밋(빌드 의존 없음). 나노바나나 목업 나오면 같은 자리 파일 교체.
- **스크린샷 신선도(운영 노트)**: `/m` 디자인이 바뀌면 스크린샷 재촬영 필요 — vault [[홈-예시노출-사진완화]]에 운영 규칙으로 기록(자동 메커니즘 없음, interim 한정 수용).

## 4. 폼 내 진입점 A′ (`src/app/register/page.tsx` — ⚠️ 핫파일)
- **메인 브랜드 사진 섹션 헤더 아래 1곳만**, 작은 텍스트 링크:
  > **"사진 없이 소개서를 만들어보셔도 좋아요 → 소개서 미리보기"** (대표 확정 문구)
- 탭하면 **바텀시트**(기존 시트 패턴 + `ScrollLock` 재사용). 닫으면 폼 스크롤 위치·포커스 그대로(시트 오버레이라 폼 언마운트 없음).
- **시트 내용 v1 = 정적 이미지**(홈 섹션과 동일 자산, 사진 없는 버전 먼저) + "전체 화면으로 보기 → /preview" 는 두지 않음(폼 이탈 방지). **iframe 실페이지 임베드는 후속 개선**으로 강등 — 정적이 3G에서 빠르고 iOS 중첩 스크롤 리스크 0, QA 사이클 절약 (레드팀 NIT-1 수용, 기본값 반전).
- 카드·블록에는 반복 노출 없음(단일 터치포인트 결정 준수). 초본완성 얼럿 코드 불변.

## 5. 사진 UI 완화 (register + BlockEditor — ⚠️ 핫파일)
- ① 브랜드 사진(최대 10장): **현행 유지**.
- 활동 카드·콜라보 카드·블록 6종 공통 첨부: 기본 상태를 **접힌 텍스트 버튼 "+ 사진 담기 (선택)"**으로 축소 → 탭하면 기존 PhotoGrid 펼침. **이미 사진이 있는 카드는 펼친 채 시작**(기존 데이터 은닉 금지). 수정 모드도 동일 규칙.
- 데이터·업로드 로직 변경 없음 — 순수 표시 계층.
