# 소개서 폼 필수 최소화 + AI 리빌 — 설계 (2026-07-14)

> 브레인스토밍(대표+1팀+상설 레드팀 7라운드) 확정본. "쉽게 시작하고, AI가 선물처럼 채워주고, 부족하면 다정하게 더 묻는" 소개서 작성 경험.
> 선행: 블록 모듈화 Phase 1·2 배포됨(`docs/superpowers/specs/2026-07-13-intro-blocks-design.md`). 기존 Phase 3(blockHints 배너)은 본 설계의 AI 리빌로 **흡수**.

## 배경 · 목표

현재 9섹션 전부가 폼에 펼쳐져 있어 진입 순간 "쓸 게 너무 많다"는 공포를 줌. 목표:
1. 첫인상 = 필수 3구역의 가벼운 폼 (온보딩 공포 제거)
2. AI 크롤 결과를 "이런 이야기들을 찾았어요!" 선물 순간으로 (리뷰형 리빌)
3. 짧은 소개서는 강제가 아니라 **설득**으로 보완 (완성 기준 + 넛지 정책)

## 확정 결정 요약

| 결정 | 내용 |
|---|---|
| 폼 구조 | **하이브리드 스텁** — 필수(①+협업칩+⑨) / 인라인 접힌 스텁 3종(②④⑦) / 시트 두 그룹(③⑧⑤서술⑥ + 블록6종) |
| ⑤(어떤 협업) | 칩만 **①로 이사, 필수 유지**(검색·매칭 하드축). 서술(offersNote)은 시트 옵션 |
| ⑥(찾는 파트너) | **통째 시트 옵션으로** — ⚠️ 초기 합의(⑤⑥ 필수 유지)의 **의식적 번복**. 결과: 매칭 신호가 offers 중심으로 반쪽. 완화책 필수(아래 §매칭 완화) |
| AI 크롤 | **리뷰형 리빌** — 자동채움 금지 정책 유지, 카드별 [담기] 원탭 |

| 카피 | **기존 9섹션 질문 문장 그대로 승계** — 새 문장 필요 시 그 글체(질문형·해요체) |
| 비주얼 | 다닥다닥 금지(섹션 리듬 51px 유지), 키위는 행동 유도 지점에만(스텁 +·시트 그룹 헤더·리빌 카드 테두리) |

## 1. 폼 재구성 (/register · 수정폼 동일)

위→아래:

| 구역 | 내용 | 상태 |
|---|---|---|
| ① 브랜드를 소개해주세요. | 상호 · 한 줄 소개 · 자세히 소개(초안 받기 유지) · 브랜드 사진 · **협업 유형 칩(라벨: "어떤 협업을 할 수 있나요?" — 구⑤ 제목 승계)** | 필수 (칩 최소 1개) |
| 스텁 A | `+ 왜 이 브랜드를 시작하셨나요?` | 접힌 한 줄 |
| 스텁 B | `+ 주로 어떤 활동을 하나요?` | 접힌 한 줄 |
| 스텁 C | `+ 이런 콜라보 경험이 있어요.` | 접힌 한 줄 |
| 강조 카드 + FAB | "브랜드의 이야기를 더 담아볼까요?" → 바텀시트 (기존 유지) | — |
| 완성도 안내 한 줄 | §3 | — |
| ⑨ 브랜드 정보를 입력해주세요. | 주소·인스타·홈피 + 소개자료 PDF(선택) | 필수 틀 = 섹션 상시 노출. 개별 필드 검증은 현행 유지(강제 없음) |

**바텀시트 두 그룹** (그룹 헤더에 키위 포인트):
- **브랜드 이야기**: `이런 파트너를 찾고 있어요.`(구⑥ 칩+서술, **맨 위 고정** — 매칭 완화책) · `우리 브랜드를 표현하는 키워드를 골라주세요.`(구③) · `저희는 주로 이런 고객과 함께하고 있어요.`(구⑧) · `어떤 협업을 할 수 있나요? — 자세히`(구⑤ 서술 offersNote)
- **더 보여주기**: 블록 6종 (기존 그대로)

**스텁 동작 규칙**:
- 탭 → 그 자리에서 기존 편집 UI로 펼침(시트 이동 없음). 다시 접기 가능.
- **펼쳤지만 빈 채 = 추가 아님** — /m에 안 나오고 `isRichIntro`에도 안 셈. 실수 탭이 폼을 영구히 늘리지 않음.
- 시트에서 추가한 섹션은 폼의 **정본 위치에 인라인**으로 삽입(펼쳐진 채).
- **수정 모드**: 데이터 있는 섹션 = 정본 순서로 펼쳐진 채 복귀(시트 출신 포함). 빈 섹션 = 접힌 스텁/시트 잔류.
- 데모 프리필(`?demo=1`) = 대표 시연 도구로 현행 유지(채워진 채 펼쳐짐). 문서화만.

## 2. AI 리빌 ("이런 이야기들을 찾았어요")

**흐름**: 위저드(이름→크롤) 완료 → **리빌 스텝**(폼 진입 직전):
1. 찾은 콘텐츠를 카드 스택으로 — 카드 = 대상 섹션 제목(기존 문장) + 내용 프리뷰 3줄 + 근거 한 줄("인스타그램에서 봤어요")
2. 카드별 **[이 내용으로 담기]** / [건너뛰기]
3. 담은 카드 → 해당 섹션이 채워진 채 펼쳐져 폼 진입. ✨aiFilled 문법 재사용("확인하고 다듬어주세요" 결)

**소스**: 기존 크롤 산출(소개글 후보·활동 힌트·콜라보 힌트) + blockHints(숫자·소개된 곳·공간·고객 이야기, 최대 2 — 2026-07-13 계획 Task 13 스키마 재사용). **seeks 추천을 리빌 최우선 순위로**(매칭 완화책).
**빈손 리빌 금지**: 찾은 게 없으면 리빌 스텝 자체를 건너뜀(지어내기 없음 정책).

**정합 규칙** (레드팀 CONFIRMED 반영):
- **후보 풀 단일화**: 소개글 후보는 위저드 경유 시 리빌에서 소비. ①의 [초안 받기]는 직접 입력 사용자용으로 존치. 어느 경로든 **사용자가 수정한 텍스트는 절대 덮어쓰지 않는다.**
- **건너뛴 카드는 소멸하지 않음**: 해당 스텁/시트 섹션을 열면 섹션 안에서 "웹에서 찾은 내용이에요" 힌트로 재등장(기존 ④⑦ 접힌 배너의 영속성을 섹션 내부로 이사). 이 규칙으로 기존 크롤힌트 배너는 **은퇴**.

## 3. 완성 기준 · 설득 UX

**공용 유틸** `isRichIntro(maker)`: 필수 3구역 충족 + 섹션 ≥2 + 서사(story/activities/collabHistory) ≥1. 폼 안내·리빌 우선순위·넛지(추후)·검색랭킹(추후) 전부 이 함수 재사용.

**폼 안내 한 줄** (제출 버튼 위, 어떤 경우도 제출 안 막음·경고 톤 금지):
- 미달: "이야기를 하나만 더 담으면, 소개서가 한층 단단해져요." + 접힌 스텁 칩 1~2개(탭 = **스크롤+포커스**, 이미 펼쳐져 있으면 포커스만)
- 충족: 키위 체크 + "충분히 멋진 소개서예요. 언제든 더 담을 수 있어요."
- 게이지·퍼센트 금지.

## 4. 넛지 정책 (구현은 다음 라운드 — 정책 정본)

- 트리거: 로그인 소유자 + 내 소개서가 얇은 상태(`!isRichIntro`)
- 노출: `/my` 카드 + 내 소개서 소유자 뷰 상단 배너. **사이트 전역 추적 얼럿은 안 함**
- 액션: [+ 소개 더하기] → edit 진입 / [24시간 안 보기](로컬 저장)
- 1순위 유도 대상 = **seeks(이런 파트너를 찾고 있어요)** — step2 매칭 론칭 전 백필

## §매칭 완화 (⑥ 옵셔널화의 대가 관리)

⑥ 번복으로 매칭 신호가 offers 중심이 됨. 완화 3종 = ①리빌에서 seeks 최우선 추천 ②시트 '브랜드 이야기' 그룹 맨 위 고정 ③넛지 1순위 타깃. **step2 매칭 개발 착수 전에 seeks 보유율 점검**을 게이트로 명시.

## 구현 단계 (각각 독립 배포)

1. **폼 재구성** — ① 칩 이사·스텁 3종·시트 두 그룹·완성도 안내·⑥ 이동·수정모드 규칙
2. **AI 리빌** — 리빌 스텝 + blockHints(enrich 확장) + 배너 은퇴 + 힌트 섹션내 이사
3. **넛지** — 다음 라운드

## 검증

단계마다 `tsc --noEmit`+`npm run build` → 데모 프리필 갱신 → **대표 실기기 QA(모바일 우선)** → 배포. 수정모드 왕복(기존 풍부한 소개서 → 펼침 복귀) 필수 확인.
